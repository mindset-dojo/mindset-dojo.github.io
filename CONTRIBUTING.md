### Review expectations

- Keep Markdown simple; favor lists and short paragraphs over heavy HTML.
- Use clear YAML front matter with `---` fences. Match the patterns in existing author cards and Insights.
- When editing layouts or includes, preserve existing section keys and front matter hooks unless your PR explains the change.

### Submission checklist

- Preview locally with `bundle exec jekyll serve` if possible, or rely on GitHub Pages build feedback after merge.
- Verify links and image paths if you include media.
- Confirm filenames and directories follow the expected slug/date patterns.

### Community norms

- Be direct, respectful, and helpful; we iterate and reflect like a dojo kata.
- Small, focused pull requests speed up review and publishing.
- The LICENSE applies to all contributions; by submitting a PR you agree to its terms.

### Support

Open an Issue if you get stuck, or start a Discussion if enabled. We welcome questions that help others cross the same threshold.
